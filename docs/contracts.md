### Server Response
#### Response Object
This object being returned to the client will be in one of two states:
- Success
- Failute

The object will have three properties:
- status: "success" | "failure"
- data: T | null
- error: null | {
  code: string,
  message: string
}

## Database
### Schemas
- Meals
- Schedules
- Memberships
- Households
- Users
#### Users
- id: Automatically generated by the database; unique; required
- name: string; required
- email: string; required; unique
- role: "admin"|"editor"|"viewer"; required
- created_at: date_time_timestamp; required
#### Households
- id: Automatically generated by the database; unique; required
- created_by: Forein key - User id; required
#### Memberships
The user_id column and household_id pair will serve as the primary key; all unqie rows.
- user_id: Foreign key - User id
- household_id: Foreign key - Household id
#### Meals
- id: Automatically generated by the database; unique; required
- title: string; unique; required
- url: string; default null
#### Schedules
- Meal_id: Foreign key - Meal id
- Start_date: date_time_timestamp; required
- End-date: date_time_timestamp; required
- created_by: Forein key - User id; required
- updated_by: Forein key - User id; defaul null
